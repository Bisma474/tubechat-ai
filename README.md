# TubeChat — YouTube AI Chatbot

Chat with any YouTube video using AI. Paste a URL, load the video, and ask anything about its content.

---

## What It Does

TubeChat fetches the transcript of any YouTube video, processes it into searchable chunks, and lets you ask questions about the video content through a chat interface. Answers are generated by a large language model using only the video's transcript as context.

---

## Features

### 1. YouTube Transcript Extraction
- Accepts any standard YouTube URL format (`youtube.com/watch?v=`, `youtu.be/`, `shorts/`, `embed/`)
- Uses the **Supadata API** to fetch the transcript automatically
- Handles **short videos** (instant response) and **long videos** (async background processing with polling up to 2 minutes)
- Displays a live status message while waiting for large videos to process

### 2. RAG Pipeline (Retrieval-Augmented Generation)
The transcript goes through a full pipeline before you can chat:

| Step | What Happens |
|------|-------------|
| Load | Transcript text is fetched from Supadata |
| Split | Text is split into chunks of 800 characters with 100-character overlap using `RecursiveCharacterTextSplitter` |
| Embed | Each chunk is converted into a vector using HuggingFace `sentence-transformers/all-MiniLM-L6-v2` |
| Index | All vectors are stored in a **FAISS** vector store for fast similarity search |
| Retrieve | When you ask a question, the top 4 most relevant chunks are retrieved |
| Augment | Retrieved chunks are combined into a context block and passed to the LLM |
| Generate | Groq LLM generates the final answer based only on that context |

A **visual pipeline bar** in the UI highlights which step is currently active during loading.

### 3. Model Selection
Choose from 4 LLMs via the Groq API in the sidebar:

| Model | ID |
|-------|----|
| Llama 3.1 8B (Fast) | `llama-3.1-8b-instant` |
| Llama 3.3 70B (Best) | `llama-3.3-70b-versatile` |
| Llama 4 Scout 17B | `meta-llama/llama-4-scout-17b-16e-instruct` |
| Qwen3 32B | `qwen/qwen3-32b` |

### 4. Chat Interface
- Ask any number of questions about the video in sequence
- Full **chat history** is displayed in the session with labeled bubbles (YOU / TUBECHAT)
- Answers are always returned in **English**, even if the video transcript is in another language (Hindi, Urdu, etc.)
- LLM temperature is set to `0.3` for focused, factual answers
- Max response length is `512` tokens

### 5. Embedded YouTube Player
Once a video is loaded, the YouTube player is embedded directly in the main area so you can watch and chat at the same time.

### 6. Sidebar Status Panel
After a video loads successfully, the sidebar shows:
- A green **Ready** status indicator
- The number of **chunks** the transcript was split into

### 7. Clear / Reset
A **Clear** button in the sidebar resets the entire session — removes the transcript, vector store, chat history, and pipeline state — so you can load a new video.

### 8. API Key Management
Both API keys (`GROQ_API_KEY` and `SUPADATA_API_KEY`) are loaded from Streamlit **secrets** — they are never exposed in the UI or the code.

### 9. Embeddings Caching
The HuggingFace embeddings model is cached using `@st.cache_resource` so it only loads once per session, avoiding repeated downloads.

## Tech Stack

| Component | Technology |
|-----------|-----------|
| UI | Streamlit |
| Transcript Fetching | Supadata API |
| Text Splitting | LangChain `RecursiveCharacterTextSplitter` |
| Embeddings | HuggingFace `sentence-transformers/all-MiniLM-L6-v2` |
| Vector Store | FAISS (via LangChain) |
| LLM | Groq API (Llama / Qwen models) |

## Installation

```bash
git clone https://github.com/Bisma474/tubechat
cd tubechat
pip install -r requirements.txt
```

### requirements.txt
```
streamlit
supadata
langchain-text-splitters
langchain-huggingface
langchain-community
langchain-core
faiss-cpu
groq
sentence-transformers
```

---

## Setup API Keys

Create a `.streamlit/secrets.toml` file:

```toml
GROQ_API_KEY = "your_groq_api_key"
SUPADATA_API_KEY = "your_supadata_api_key"
```

Get your keys from:
- Groq: https://console.groq.com
- Supadata: https://supadata.ai

---

## Run the App

```bash
streamlit run app.py
```

---

## How to Use

1. Paste a YouTube URL in the sidebar
2. Select a Groq model
3. Click **Load Video**
4. Wait for the pipeline to complete (status shown in progress bar)
5. Type your question and click **Send**

---

## Limitations

- Only processes videos that have an available transcript (auto-generated or manual captions)
- Answers are based strictly on the transcript — visual content in the video is not analysed
- Large videos may take up to 2 minutes to process via the async Supadata job
- Answers are always in English regardless of the transcript language
